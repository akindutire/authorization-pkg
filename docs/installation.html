Installation Guide

Get started with model-agnostic authorization in your Laravel application. This package works with any Eloquent model, not just users.

Model-Agnostic: Unlike traditional packages, you can attach abilities to Articles, TeamMembers, Organizations, or any entity in your application.

Requirements

  • PHP: 8.1 or higher
  • Laravel: 9.0 or higher (supports Laravel 9, 10, 11+)
  • Cache: Redis or Memcached (recommended for production)

Step 1: Install via Composer

Install the package using Composer:

composer require akindutire/authorization-pkg

The package will automatically register via Laravel's package discovery.

Step 2: Publish Configuration

Publish the configuration file to customize package settings:

php artisan vendor:publish --tag=authorization-config

This creates config/akindutire-authorization.php where you can configure:

  • Ability templates for different roles
  • Custom column names
  • Cache TTL settings
  • Entity tables to index
  • Properties for fast lookups

Step 3: Database Setup

Generate Migrations for Each Entity

Create migrations for each table that will have abilities:

# For articles
php artisan make:permission-migration articles

# For team members
php artisan make:permission-migration team_members

# For organizations
php artisan make:permission-migration organizations

# For users (if needed)
php artisan make:permission-migration users

Each migration adds two JSON columns with database-specific optimizations:

  • allowed_permissions - Abilities the entity has
  • revoked_permissions - Explicitly denied abilities
Important: Generate a separate migration for each entity table. The command creates a timestamped migration customized for the specified table.

Run Migrations

Apply all migrations to your database:

php artisan migrate

Step 4: Configure Your Models

Add the HasPermissions trait to any model that needs abilities:

Articles

use Akindutire\Authorization\Traits\HasPermissions;
use Illuminate\Database\Eloquent\Model;

class Article extends Model
{
    use HasPermissions;

    protected $casts = [
        'allowed_permissions' => 'array',
        'revoked_permissions' => 'array',
    ];
}

Team Members

class TeamMember extends Model
{
    use HasPermissions;

    protected $casts = [
        'allowed_permissions' => 'array',
        'revoked_permissions' => 'array',
    ];
}

Organizations

class Organization extends Model
{
    use HasPermissions;

    protected $casts = [
        'allowed_permissions' => 'array',
        'revoked_permissions' => 'array',
    ];
}
Optional Casts: The trait automatically adds these casts if not explicitly defined. Adding them explicitly provides better IDE support.

Step 5: Register Middleware

Add the middleware to app/Http/Kernel.php:

protected $middlewareAliases = [
    // ... other middleware
    'validate.subject.action' => \Akindutire\Authorization\Middleware\ValidateSubjectAction::class,
];

Apply to route groups:

Route::middleware(['validate.subject.action'])->group(function () {
    Route::put('/articles/{id}', [ArticleController::class, 'update']);
    Route::post('/teams/{member_id}/invite', [TeamController::class, 'invite']);
});

Step 6: Configure Production Caching (Optional)

For production environments, configure Redis and warm the cache:

# .env
CACHE_DRIVER=redis
REDIS_HOST=127.0.0.1
REDIS_PORT=6379
PERMISSION_CACHE_TTL=300

Warm cache during deployment:

php artisan permission:cache

Step 7: Update Configuration

Edit config/akindutire-authorization.php to list all entity tables:

return [
    'abilities' => [
        'article_editor' => ['can_be_edited', 'can_be_deleted'],
        'team_admin' => ['can_invite', 'can_manage_billing'],
        'org_premium' => ['can_use_api', 'can_export_data'],
    ],

    'indexed_tables' => [
        'articles',
        'team_members',
        'organizations',
        'users',
    ],

    'indexed_properties' => [
        'uuid',
        'slug',
        'tenant_id',
    ],

    'cache_keys' => [
        'uuid',
        'slug',
        'tenant_id',
    ],
];

Verify Installation

Test that everything works by checking abilities in tinker:

php artisan tinker

// Test with Article
>>> $article = Article::first();
>>> $article->grantPermission(['can_be_edited', 'can_be_published']);
>>> $article->hasPermission('can_be_edited');
=> true

// Test with TeamMember
>>> $member = TeamMember::first();
>>> $member->grantPermission('can_invite');
>>> $member->hasPermission('can_invite');
=> true

// Test with Organization
>>> $org = Organization::first();
>>> $org->grantPermission(['can_use_api']);
>>> $org->hasPermission('can_use_api');
=> true

// Verify caching
>>> Cache::has('entity.App.Models.Article.id.' . $article->id);
=> true  // Cache is working

Common Installation Issues

Migration Already Exists

If you get "migration already exists", delete the old migration and regenerate:

rm database/migrations/*_add_permissions_to_articles_table.php
php artisan make:permission-migration articles

Trait Not Found

Ensure autoload is updated:

composer dump-autoload

Cache Not Working

Verify Redis is running:

redis-cli ping  # Should return PONG
