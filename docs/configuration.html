Configuration

Customize the authorization package to fit your application's needs.

Publish the configuration file:

php artisan vendor:publish --tag=akindutire-authorization-config

This creates config/akindutire-authorization.php with all available options.

Abilities Configuration

Define available permissions for your application. Two formats are supported:

Option 1: Simple Permission List

Recommended for most applications. Define a flat list of available permissions:

'abilities' => [
    'can_edit',
    'can_delete',
    'can_broadcast',
    'can_autosend_for_analytics',
],

Option 2: Role-Based Permission Groups

Define role templates for reference and documentation purposes:

'abilities' => [
    'owner' => [
        'can_update_company',
        'can_invite_member',
        'can_remove_member',
    ],
    'admin' => [
        'can_invite_member',
        'can_view_pitch',
    ],
    'member' => [
        'can_view_pitch',
    ],
],
Note: These are informational templates only. Access them via EntityPermission::getAbilities(). You must still explicitly grant permissions using grantPermission().

Example:

// Get admin permissions from config
$adminPerms = EntityPermission::getAbilities()['admin'];
// Then explicitly grant them
$user->grantPermission($adminPerms);

Column Names

Customize the database column names used for storing permissions:

'column_names' => [
    'allowed_permissions' => 'allowed_permissions',
    'revoked_permissions' => 'revoked_permissions',
],

Different entity types can use different column names:

// For users
'allowed_permissions' => 'allowed_permissions'

// For articles (customize per model if needed)
'allowed_permissions' => 'capabilities'
'revoked_permissions' => 'restrictions'
Important: Ensure your migration creates columns matching these names, and update model configurations accordingly.

Cache Configuration

Entity Cache TTL

Controls how long entity lookups are cached (in seconds):

'entity_cache_ttl' => env('PERMISSION_CACHE_TTL', 300),
Value Effect Use Case
60-300 Balanced performance Most applications
600-3600 Maximum performance Read-heavy workloads
0-30 Near real-time updates Frequent permission changes

Override in .env:

PERMISSION_CACHE_TTL=300

Auto-Invalidate Reflection Cache

Automatically detect when attributes change and update caches:

'auto_invalidate_reflection_cache' => env('PERMISSION_AUTO_INVALIDATE', true),
Setting Behavior Recommended For
true Cache keys include attribute hash Development & Production
false Cache persists until manual clear Legacy behavior only
Recommended: Keep this enabled (true). It eliminates the need to manually clear caches when updating attributes, while adding only ~5-10μs overhead on first request.

Override in .env:

PERMISSION_AUTO_INVALIDATE=true

Cache Keys

Properties to index for cache invalidation:

'cache_keys' => [
    'uuid',
    'email',
    'slug',
    // Add other properties used for entity lookups
],

Add any properties used in your #[HasAny] or #[HasAll] attributes:

// If you use:
#[HasAny($actions, User::class, 'username')]

// Add to cache_keys:
'cache_keys' => [
    'uuid',
    'email',
    'slug',
    'username', // ← Add this
],

Database Index Configuration

Indexed Tables

Tables that use the HasPermissions trait:

'indexed_tables' => [
    'users',
    'articles',
    'team_members',
    'posts',
],
Performance: Adding tables to this list enables performance indexes. Critical for applications with >1M rows.

Indexed Properties

Properties commonly used in WHERE clauses and attribute lookups:

'indexed_properties' => [
    'uuid',
    'email',
    'slug',
    // Add properties used in #[HasAny(..., Model::class, 'property')]
],

These should match properties used in your attributes:

#[HasAny($actions, User::class, 'uuid')]
                                     ^^^^^^
                                     Add to indexed_properties

Permission Size Limits

Max Permission Size (Bytes)

Maximum size of permission JSON in bytes:

'max_permission_size_bytes' => env('PERMISSION_MAX_SIZE_BYTES', 10240), // 10KB
Value Approx. Permissions Use Case
10240 (10KB) 250-400 permissions Default (recommended)
5120 (5KB) 125-200 permissions Memory-constrained
null Unlimited Disable validation

Override in .env:

PERMISSION_MAX_SIZE_BYTES=10240

Max Permission Count

Maximum number of individual permissions per entity:

'max_permission_count' => env('PERMISSION_MAX_COUNT', 500),

Override in .env:

PERMISSION_MAX_COUNT=500
Need more than 500 permissions? Consider:
  • Permission namespacing (e.g., 'article.edit', 'article.delete')
  • Bitwise permission encoding for ultra-compact storage
  • External permission storage (dedicated permissions table)

Exception Handling

Configure how authorization exceptions should be handled:

'exception' => [
    'message' => env(
        'PERMISSION_EXCEPTION_MESSAGE',
        'Access denied, you do not have enough permission to perform this action, contact your administrator'
    ),
    'code' => 403,
],

Customize the error message in .env:

PERMISSION_EXCEPTION_MESSAGE="You don't have permission to do that."

Environment Variables

Summary of all available environment variables:

# Cache configuration
PERMISSION_CACHE_TTL=300
PERMISSION_AUTO_INVALIDATE=true

# Size limits
PERMISSION_MAX_SIZE_BYTES=10240
PERMISSION_MAX_COUNT=500

# Exception handling
PERMISSION_EXCEPTION_MESSAGE="Access denied, contact your administrator"

Configuration Examples

High-Performance Production

// config/akindutire-authorization.php
'entity_cache_ttl' => 3600, // 1 hour
'auto_invalidate_reflection_cache' => true,

// .env
PERMISSION_CACHE_TTL=3600
PERMISSION_AUTO_INVALIDATE=true

Real-Time Permissions

// config/akindutire-authorization.php
'entity_cache_ttl' => 30, // 30 seconds
'auto_invalidate_reflection_cache' => true,

// .env
PERMISSION_CACHE_TTL=30
PERMISSION_AUTO_INVALIDATE=true

Development Environment

// config/akindutire-authorization.php
'entity_cache_ttl' => 0, // No caching
'auto_invalidate_reflection_cache' => true,

// .env
PERMISSION_CACHE_TTL=0
PERMISSION_AUTO_INVALIDATE=true
Tip: Use different values per environment by setting them in your .env files rather than modifying the config file directly.
