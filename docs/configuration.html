Configuration

Customize the authorization package to fit your application's needs. Configure ability templates, caching strategies, and performance optimizations.

Model-Agnostic: All configuration options work with any Eloquent model (Article, TeamMember, Organization, etc.). Configure abilities for each entity type independently.

Publishing Configuration

Publish the configuration file to customize package settings:

php artisan vendor:publish --tag=authorization-config

This creates config/akindutire-authorization.php with all available options.

Ability Templates

Define reusable ability templates for different entity types and roles.

Entity-Specific Ability Templates

Recommended approach: Define templates per entity type for better organization.

'abilities' => [
    // Article abilities
    'article_editor' => [
        'can_be_edited',
        'can_be_deleted',
    ],
    'article_publisher' => [
        'can_be_edited',
        'can_be_published',
        'can_be_featured',
    ],

    // Team member abilities
    'team_admin' => [
        'can_invite',
        'can_remove',
        'can_manage_billing',
    ],
    'team_member' => [
        'can_invite',
        'can_view_analytics',
    ],

    // Organization abilities
    'org_free' => [
        'can_create_projects',
    ],
    'org_premium' => [
        'can_create_projects',
        'can_use_api',
        'can_export_data',
    ],
    'org_enterprise' => [
        'can_create_projects',
        'can_use_api',
        'can_export_data',
        'can_white_label',
        'can_sso',
    ],
],
Note: These are templates only. You must explicitly grant them to entities using EntityPermission::getAbilities('template_name').

Usage Examples:

use Akindutire\Authorization\Facades\EntityPermission;

// Grant article publisher abilities
$article = Article::find(1);
$article->grantPermission(EntityPermission::getAbilities('article_publisher'));

// Grant team admin abilities
$member = TeamMember::find(1);
$member->grantPermission(EntityPermission::getAbilities('team_admin'));

// Grant enterprise organization abilities
$org = Organization::find(1);
$org->grantPermission(EntityPermission::getAbilities('org_enterprise'));

Simple Ability List

Alternative: Define a flat list for simple applications.

'abilities' => [
    'can_be_edited',
    'can_be_deleted',
    'can_be_published',
    'can_invite',
    'can_manage_billing',
],

Column Names

Customize database column names for storing abilities:

'column_names' => [
    'allowed_permissions' => 'allowed_permissions',  // Granted abilities
    'revoked_permissions' => 'revoked_permissions',  // Denied abilities
],

You can use different column names if needed:

'column_names' => [
    'allowed_permissions' => 'abilities',
    'revoked_permissions' => 'denied_abilities',
],
Important: Ensure your migrations create columns matching these names, and update model $casts accordingly.

Cache Configuration

Entity Cache TTL

Controls how long entity ability lookups are cached (in seconds):

'entity_cache_ttl' => env('PERMISSION_CACHE_TTL', 300),  // 5 minutes
Value Effect Use Case
60-300 Balanced performance Most applications (recommended)
600-3600 Maximum performance Read-heavy workloads
0-30 Near real-time updates Frequent ability changes
0 No caching Development only

Override in .env:

PERMISSION_CACHE_TTL=300
Auto-Invalidation: Caches automatically clear when entity abilities change via grantPermission() or revokePermission(). You don't need to manually clear caches.

Reflection Cache

Enable automatic reflection metadata cache invalidation:

'reflection_cache_enabled' => true,
'auto_invalidate_reflection_cache' => env('PERMISSION_AUTO_INVALIDATE', true),
Setting Behavior Recommended For
true Cache keys include attribute hash Development & Production
false Cache persists until manual clear Legacy behavior only
Recommended: Keep this enabled. It eliminates manual cache clearing when updating attributes, adding only ~5-10μs overhead.

Override in .env:

PERMISSION_AUTO_INVALIDATE=true

Cache Keys

Properties to track for multi-key cache invalidation:

'cache_keys' => [
    'id',
    'uuid',
    'slug',
    'email',
    'tenant_id',
],

Add any properties used in your attribute lookups:

// If you use these attributes:
#[HasAny(['can_be_edited'], Article::class, 'slug')]
#[HasAny(['can_invite'], TeamMember::class, 'uuid')]
#[HasAny(['can_use_api'], Organization::class, 'tenant_id')]

// Add to cache_keys:
'cache_keys' => [
    'id',      // Always include
    'uuid',    // For TeamMember lookups
    'slug',    // For Article lookups
    'tenant_id', // For Organization lookups
],
Important: When abilities change, all cache entries for these properties are automatically invalidated for that entity.

Database Indexing

Indexed Tables

Tables that use the HasPermissions trait:

'indexed_tables' => [
    'articles',
    'team_members',
    'organizations',
    'users',
],
Performance: Adding tables enables query optimizations. Critical for tables with >1M rows.

Indexed Properties

Properties commonly used for entity lookups:

'indexed_properties' => [
    'uuid',
    'slug',
    'email',
    'tenant_id',
],

Match these to properties used in your attributes:

#[HasAny(['can_be_edited'], Article::class, 'slug')]
                                              ^^^^^^
                                              Add 'slug' to indexed_properties

#[HasAny(['can_use_api'], Organization::class, 'tenant_id')]
                                                 ^^^^^^^^^^^
                                                 Add 'tenant_id' to indexed_properties

Size Limits

Max Permission Size

Maximum size of abilities JSON in bytes:

'max_permission_size_bytes' => env('PERMISSION_MAX_SIZE_BYTES', 10240), // 10KB
Value Approx. Abilities Use Case
10240 (10KB) 250-400 abilities Default (recommended)
5120 (5KB) 125-200 abilities Memory-constrained
null Unlimited Disable validation

Override in .env:

PERMISSION_MAX_SIZE_BYTES=10240

Max Permission Count

Maximum number of individual abilities per entity:

'max_permission_count' => env('PERMISSION_MAX_COUNT', 500),

Override in .env:

PERMISSION_MAX_COUNT=500
Need more than 500 abilities per entity? Consider:
  • Namespacing abilities (e.g., articles:edit, articles:delete)
  • Splitting entity types (separate Article and Draft models)
  • External permission storage (dedicated permissions table)

Exception Handling

Configure how authorization failures are handled:

'exception' => [
    'message' => env(
        'PERMISSION_EXCEPTION_MESSAGE',
        'Access denied: insufficient permissions to perform this action'
    ),
    'code' => 403,
],

Customize the error message in .env:

PERMISSION_EXCEPTION_MESSAGE="You don't have permission to access this resource."

Example: Custom exception handling in your app:

// app/Exceptions/Handler.php
use Illuminate\Auth\Access\AuthorizationException;

public function render($request, Throwable $exception)
{
    if ($exception instanceof AuthorizationException) {
        return response()->json([
            'error' => 'unauthorized',
            'message' => config('akindutire-authorization.exception.message'),
        ], 403);
    }

    return parent::render($request, $exception);
}

Environment Variables

All configuration can be overridden via environment variables:

# .env

# Cache configuration
CACHE_DRIVER=redis
PERMISSION_CACHE_TTL=300
PERMISSION_AUTO_INVALIDATE=true

# Size limits
PERMISSION_MAX_SIZE_BYTES=10240
PERMISSION_MAX_COUNT=500

# Exception handling
PERMISSION_EXCEPTION_MESSAGE="Access denied: contact administrator"

Configuration Examples

High-Performance Production

Optimized for maximum throughput with read-heavy workloads:

// config/akindutire-authorization.php
return [
    'entity_cache_ttl' => 600, // 10 minutes
    'auto_invalidate_reflection_cache' => true,
    'reflection_cache_enabled' => true,

    'indexed_tables' => [
        'articles',
        'team_members',
        'organizations',
    ],

    'cache_keys' => [
        'id',
        'uuid',
        'slug',
    ],
];

// .env
CACHE_DRIVER=redis
REDIS_HOST=127.0.0.1
REDIS_PORT=6379
PERMISSION_CACHE_TTL=600
PERMISSION_AUTO_INVALIDATE=true

Expected Performance:

  • Cache hit rate: 99%+
  • Authorization check: <1ms (p50)
  • Database queries: 98% reduction

Real-Time Permissions

For applications requiring immediate ability updates:

// config/akindutire-authorization.php
return [

    'auto_invalidate_reflection_cache' => true,
];

// .env
PERMISSION_CACHE_TTL=30
PERMISSION_AUTO_INVALIDATE=true

Development Environment

Minimal caching for rapid development:

// config/akindutire-authorization.php
return [

// .env
CACHE_DRIVER=array
PERMISSION_CACHE_TTL=0
PERMISSION_AUTO_INVALIDATE=true

Multi-Tenant SaaS

Configuration for multi-tenant applications:

// config/akindutire-authorization.php
return [
    'abilities' => [
        'org_free' => ['can_create_projects'],
        'org_premium' => ['can_create_projects', 'can_use_api'],
        'org_enterprise' => ['can_create_projects', 'can_use_api', 'can_sso'],
    ],

    'cache_keys' => [
        'id',
        'uuid',
        'tenant_id', // Important for tenant isolation
    ],

    'indexed_tables' => [
        'organizations',
        'team_members',
    ],

    'indexed_properties' => [
        'uuid',
        'tenant_id',
    ],
];

// Usage
$org = Organization::find(1);
$org->grantPermission(EntityPermission::getAbilities('org_enterprise'));

Complete Configuration File

Full example with all options:

// config/akindutire-authorization.php

return [
    // Ability templates
    'abilities' => [
        'article_editor' => ['can_be_edited', 'can_be_deleted'],
        'article_publisher' => ['can_be_edited', 'can_be_published', 'can_be_featured'],
        'team_admin' => ['can_invite', 'can_remove', 'can_manage_billing'],
        'org_premium' => ['can_use_api', 'can_export_data'],
    ],

    // Column names
    'column_names' => [
        'allowed_permissions' => 'allowed_permissions',
        'revoked_permissions' => 'revoked_permissions',
    ],

    // Cache settings
    'entity_cache_ttl' => env('PERMISSION_CACHE_TTL', 300),
    'reflection_cache_enabled' => true,
    'auto_invalidate_reflection_cache' => env('PERMISSION_AUTO_INVALIDATE', true),

    // Cache invalidation keys
    'cache_keys' => [
        'id',
        'uuid',
        'slug',
        'tenant_id',
    ],

    // Database indexing
    'indexed_tables' => [
        'articles',
        'team_members',
        'organizations',
        'users',
    ],

    'indexed_properties' => [
        'uuid',
        'slug',
        'tenant_id',
    ],

    // Size limits
    'max_permission_size_bytes' => env('PERMISSION_MAX_SIZE_BYTES', 10240),
    'max_permission_count' => env('PERMISSION_MAX_COUNT', 500),

    // Exception handling
    'exception' => [
        'message' => env(
            'PERMISSION_EXCEPTION_MESSAGE',
            'Access denied: insufficient permissions'
        ),
        'code' => 403,
    ],
];
Best Practice: Use environment-specific .env files to override config values rather than modifying the config file directly. This keeps your configuration portable across environments.
