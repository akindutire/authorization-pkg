API Reference

Complete reference for all attributes, traits, and services.

Attributes

#[HasAny]

Requires the user to have at least ONE of the specified permissions.

#[HasAny(
    array $actions,              // Required: Array of permission names
    ?string $model = null,       // Optional: Eloquent model class
    ?string $modelColumn = null  // Optional: Column linking resource to user
)]

Example:

#[HasAny(['can_view', 'can_edit'], Post::class, 'user_id')]
public function show(#[SubjectValue('id')] int $id)
{
    // User needs either 'can_view' OR 'can_edit' permission
}

#[HasAll]

Requires the user to have ALL of the specified permissions.

#[HasAll(
    array $actions,              // Required: Array of permission names
    ?string $model = null,       // Optional: Eloquent model class
    ?string $modelColumn = null  // Optional: Column linking resource to user
)]

Example:

#[HasAll(['can_update', 'can_delete'], Post::class, 'user_id')]
public function destroy(#[SubjectValue('id')] int $id)
{
    // User needs both 'can_update' AND 'can_delete' permissions
}

#[SubjectValue]

Marks a controller parameter as the resource identifier for permission checking.

#[SubjectValue(string $parameterName)]

Example:

#[HasAny(['can_view'], Post::class, 'user_id')]
public function show(#[SubjectValue('id')] int $postId)
{
    // The 'id' route parameter is used to fetch the Post model
}

HasPermissions Trait

Add this trait to any Eloquent model that needs permission management.

grantPermission()

Grants permissions to the model using atomic database operations.

public function grantPermission(string|array $permissions): bool

Examples:

// Grant multiple permissions
$user->grantPermission(['can_view', 'can_edit']);

// Grant single permission
$user->grantPermission('can_delete');

revokePermission()

Explicitly denies permissions. Revoked permissions override allowed permissions.

public function revokePermission(string|array $permissions): bool

Examples:

// Revoke individual permissions
$user->revokePermission(['can_delete']);

// Revoke single permission
$user->revokePermission('can_update');

hasPermission()

Checks if the model has a specific permission (accounting for revocations).

public function hasPermission(string $permission): bool

Examples:

if ($user->hasPermission('can_edit')) {
    // User has permission
}

// Revoked permissions return false
$user->grantPermission(['can_view', 'can_edit']);
$user->revokePermission(['can_edit']);
$user->hasPermission('can_edit'); // false

hasAllPermissions()

Checks if the model has all of the specified permissions.

public function hasAllPermissions(array $permissions): bool

Examples:

if ($user->hasAllPermissions(['can_update', 'can_delete'])) {
    // User has both permissions
}

hasAnyPermission()

Checks if the model has at least one of the specified permissions.

public function hasAnyPermission(array $permissions): bool

Examples:

if ($user->hasAnyPermission(['can_view', 'can_edit'])) {
    // User has at least one permission
}

EntityPermission Facade

Fluent facade for permission checking against specific models.

subject()

Sets the subject (resource) to check permissions against.

public function subject(object $subject): PermissionSvc

Examples:

use Akindutire\Authorization\Facades\EntityPermission;

$post = Post::find($id);
EntityPermission::subject($post)
    ->hasAny(['can_update']);

hasAny()

Checks if the subject has ANY of the specified permissions.

public function hasAny(array $actions): bool

Examples:

$canAccess = EntityPermission::subject($post)
    ->hasAny(['can_view', 'can_edit']);

if ($canAccess) {
    // Subject can either view or edit
}

hasAll()

Checks if the subject has ALL of the specified permissions.

public function hasAll(array $actions): bool

Examples:

$canDestroy = EntityPermission::subject($post)
    ->hasAll(['can_update', 'can_delete']);

if ($canDestroy) {
    // Subject can both update and delete
}

getAbilities()

Retrieves the ability templates from config.

public function getAbilities(): array

Examples:

// Get all abilities configuration
$abilities = EntityPermission::getAbilities();
// Returns: ['owner' => [...], 'admin' => [...], ...]

Artisan Commands

make:permission-migration

Generates a migration file to add permission columns to a table.

php artisan make:permission-migration {table}

Example:

php artisan make:permission-migration users

# Creates migration: xxxx_xx_xx_add_permission_fields_to_users_table.php

permission:cache

Pre-warms reflection metadata cache for all routes. Run during deployment for optimal performance.

php artisan permission:cache

Output:

Warming permission reflection caches...
✓ Permission cache warmed successfully
  Cached: 45 routes
  Skipped: 12 routes

permission:cache-clear

Clears all permission-related caches (reflection metadata and entity permissions).

php artisan permission:cache-clear

Output:

Clearing permission caches...
✓ Permission caches cleared successfully
  Reflection cache cleared: 45 keys
  Entity cache cleared: 128 keys
Auto-Invalidation: When enabled in config, reflection caches automatically update when attributes change. Cache clearing is only needed when disabling auto-invalidation or troubleshooting.

Middleware

ValidateSubjectAction

Automatically validates permissions based on controller method attributes.

use Akindutire\Authorization\Middleware\ValidateSubjectAction;

Route::middleware([ValidateSubjectAction::class])->group(function () {
    Route::resource('posts', PostController::class);
});

Behavior:

  • Parses #[HasAny] and #[HasAll] attributes from controller methods
  • Caches reflection metadata for performance (sub-microsecond for cached checks)
  • Automatically fetches the subject model using #[SubjectValue] parameter
  • Returns 403 Forbidden if user lacks required permissions
  • Passes request through if authorized

Performance Characteristics

Operation Performance Notes
Cached reflection lookup <1μs Sub-microsecond for warmed cache
First reflection parse ~100-200μs One-time cost per route
Permission check ~10-20μs Simple array operations
Database query ~1-5ms Standard Eloquent query
Production Ready: With warmed caches, authorization checks add negligible overhead to your application (typically <1ms per request).
