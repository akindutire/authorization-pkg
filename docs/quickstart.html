Quick Start Guide

Learn how to implement authorization in your Laravel application in 5 minutes.

Step 1: Define Model abilities/properties grouped by tags(Optional)

A user, post, campaigns, ..etc

Optionally configure ability templates in config/akindutire-authorization.php for reference:

'abilities' => [
    'owner' => ['can_update', 'can_delete', 'can_create', 'can_view'],
    'admin' => ['can_update', 'can_create', 'can_view'],
    'member' => ['can_view'],
    'virality' => ['can_go_viral', 'can_boost']
],

Note: These are informational templates only. Access them via EntityPermission::getAbilities('tag'). They are not automatically applied.

Step 2: Grant Permissions/traits/properties

Use the HasPermissions trait methods to manage permissions:

// Grant permissions to a user
$user->grantPermission(['can_view', 'can_edit']);

// Grant a single permission
$user->grantPermission('can_delete');

// Revoke permissions (overrides allowed permissions)
$user->revokePermission(['can_delete']);

Step 3: To effectively attach identifier of model (for instace post) to validate, you can attach post id via a middleware or route parameter. Protect Routes with ValidateSubjectAction Middleware

Add the current.subject.action middleware to your routes:

Add the validate.subject.action middleware to your routes:

use Akindutire\Authorization\Middleware\ValidateSubjectAction;

Route::middleware([ValidateSubjectAction::class])->group(function () {
    Route::get('/posts/{id}', [PostController::class, 'show']);
    Route::put('/posts/{id}', [PostController::class, 'update']);
    Route::delete('/posts/{id}', [PostController::class, 'destroy']);
});

Step 4: Add Attributes to Controllers

Use PHP 8 attributes to declaratively define authorization requirements:

use Akindutire\Authorization\Attributes\{HasAny, HasAll, SubjectValue};

class PostController extends Controller
{
    #[HasAny(['can_view'], User::class, 'id')]
    public function show(#[SubjectValue('user_id')] Request $request, int $id)
    {
        $post = Post::findOrFail($id);
        return view('posts.show', compact('post'));
    }

    #[HasAll(['can_update'], Post::class, 'user_id')]
    public function update(
        Request $request,
        #[SubjectValue('id')] int $id
    ) {
        $post = Post::findOrFail($id);
        $post->update($request->validated());
        return redirect()->route('posts.show', $post);
    }

    #[HasAll(['can_delete'], Post::class, 'user_id')]
    public function destroy(#[SubjectValue('id')] int $id)
    {
        Post::findOrFail($id)->delete();
        return redirect()->route('posts.index');
    }
}

Understanding the Attributes

  • #[HasAny(['can_view'])] - User needs at least ONE of the listed permissions
  • #[HasAll(['can_update', 'can_delete'])] - User needs ALL listed permissions
  • Post::class - The Eloquent model to check permissions against
  • 'user_id' - The column name that links the resource to the user
  • #[SubjectValue('id')] - The route parameter containing the resource ID

Step 5: Check Permissions Programmatically

For complex authorization logic, use the facade or model methods:

use Akindutire\Authorization\Facades\EntityPermission;

// Check if user has permission
if ($user->hasPermission('can_edit')) {
    // Allow editing
}

// Check multiple permissions (requires ALL)
if ($user->hasAllPermissions(['can_update', 'can_delete'])) {
    // Allow destructive actions
}

// Check using the facade
$post = Post::find($id);
$canEdit = EntityPermission::subject($post)
    ->hasAny(['can_update']);

if ($canEdit) {
    // Allow editing this specific post
}

Step 6: Handle Unauthorized Access

The middleware automatically returns a 403 response for unauthorized users. Customize the response in your exception handler:

// app/Exceptions/Handler.php
public function render($request, Throwable $exception)
{
    if ($exception instanceof AuthorizationException) {
        return response()->json([
            'message' => 'You do not have permission to perform this action.'
        ], 403);
    }

    return parent::render($request, $exception);
}

Complete Example: Blog System

// 1. Grant permissions to user
$user->grantPermission(['can_create', 'can_update', 'can_delete', 'can_view']);

// 2. Protect routes
Route::middleware([ValidateSubjectAction::class])->group(function () {
    Route::resource('posts', PostController::class);
});

// 3. Add attributes to controller
class PostController extends Controller
{
    #[HasAll(['can_create'])]
    public function store(Request $request)
    {
        $post = auth()->user()->posts()->create($request->validated());
        return redirect()->route('posts.show', $post);
    }

    #[HasAny(['can_view'], Post::class, 'user_id')]
    public function show(#[SubjectValue('id')] int $id)
    {
        $post = Post::with('user')->findOrFail($id);
        return view('posts.show', compact('post'));
    }

    #[HasAll(['can_update'], Post::class, 'user_id')]
    public function update(
        Request $request,
        #[SubjectValue('id')] int $id
    ) {
        $post = Post::findOrFail($id);
        $post->update($request->validated());
        return redirect()->route('posts.show', $post);
    }
}
That's it! You now have a fully functional authorization system. The middleware automatically checks permissions, caches reflection metadata, and handles unauthorized access.
